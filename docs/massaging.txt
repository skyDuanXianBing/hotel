# Messaging API

Messaging refers to the direct communication between properties and the guests through the messages. With the help of messaging, customers with or without booking can ask questions or communicate to the properties.

{% hint style="info" %}
Su Messaging API helps you to collect guests' messages from the OTA and respond to them.

* This feature enables properties to communicate with guests rather than having to login to channel extranet.
* Messages can be from guest with a booking or without a booking.
* Only available for supported channel(s).
* Only available via API.
  {% endhint %}

***

## <mark style="color:yellow;">Collection of Messages from OTA</mark>

The messages received from OTA are delivered to partner by post method. Messages are posted to an endpoint provided by partner. Please note messages can only be delivered to a single dedicated endpoint. Upon receipt, partners will need to filter messages based on property ID.

### <mark style="color:yellow;">Endpoint</mark>

**END POINT:** Please provide endpoint for messages delivery.

**Authorization String:** Partner to allocate with Basic Authentication credentials.

#### <mark style="color:yellow;">Attributes</mark>

***

`message` <mark style="color:orange;">**string (Required)**</mark>

The original message content received from OTA.

***

`guestid` <mark style="color:orange;">**string (Required)**</mark>

The unique guest ID as assigned by the provider (OTA) upon creation.

> ID can be both numeric, alphanumeric and may have special character such as hyphen ( - ).

***

`bookingid` <mark style="color:orange;">**string (Required)**</mark>

Booking id of the customer’s booking.

> Booking id can be both numeric, alphanumeric and may have special character such as hyphen ( - ).&#x20;
>
> For Airbnb, if communication started before booking is made, the parameter will be the thread ID.

***

`listingid` <mark style="color:orange;">**string (Required)**</mark>

Property id of the hotel at the time of listing (Channel Hotel ID).

***

`bookingflag` <mark style="color:orange;">**string (Required)**</mark>

The flag of the booking.

> Values can be either B or T (B = Booking, T = Thread).&#x20;
>
> Note if there is no booking ID, the value will the same for both booking and thread ID.

***

`messageid` <mark style="color:orange;">**string (Required)**</mark>

Message id of the message delivered from the OTA. Generated while delivering it from OTA.

> ID can be both numeric, alphanumeric and may have special character such as hyphen ( - ).

***

`channel_id` <mark style="color:orange;">**string (Required)**</mark>

Id of the channel through which the message delivered. Generated while delivering message from OTA.

> Airbnb Channel ID: 244\
> Booking.com ID: 19
>
> Expedia: 9

***

`threadid` <mark style="color:orange;">**string (Required)**</mark>

The thread id created when communication takes place between guest and client.

> ID can be both numeric, alphanumeric and may have special character such as hyphen ( - ).

***

#### <mark style="color:purple;">user\_details object</mark>

This object contains all the details of the users and their roles. There are two arrays users and roles in this object.

It is the user ID assigned by OTA to both Guest and Host communicating. It includes the details on user role and id. Role can be Owner, guest, co host, etc.

{% hint style="info" %}
OTA User Role and ID <mark style="color:orange;">**(only allocated for Airbnb**</mark><mark style="color:yellow;">**)**</mark>
{% endhint %}

***

#### <mark style="color:blue;">roles array</mark>

roles array contains all the different roles and the details which are involved in the messaging.

#### <mark style="color:yellow;">Attributes</mark>

***

`role` <mark style="color:orange;">**string (Required)**</mark>

Role of the user.

***

`user_ids` <mark style="color:orange;">**string (Required)**</mark>

User id of the specific role and user combination.

***

#### <mark style="color:blue;">users array</mark>

users array contains all the user and their details such as name, location, preferred\_locale and ids.&#x20;

{% hint style="info" %}
OTA User details <mark style="color:orange;">**(only allocated for Airbnb)**</mark>
{% endhint %}

#### <mark style="color:yellow;">Attributes</mark>

***

`first_name` <mark style="color:orange;">**string (Required)**</mark>

First name of the user.

***

`location` <mark style="color:orange;">**string (Required)**</mark>

Location of the user.

***

`preferred_locale` <mark style="color:orange;">**string (Required)**</mark>

Preferred language of the user.

***

`id` <mark style="color:orange;">**string (Required)**</mark>

Id of the user and role combination.

#### <mark style="color:purple;">booking\_details object</mark>

This object contains all the details of the booking for which the messaging is initiated by either property or guest. Details allocated from OTA for which guest looking for and communicating with host.

{% hint style="info" %}
OTA booking details <mark style="color:orange;">**(only allocated for Airbnb)**</mark>
{% endhint %}

#### <mark style="color:yellow;">Attributes</mark>

***

`expected_payout_amount_accurate` <mark style="color:orange;">**string (Required)**</mark>

Accurate amount which is expected to be paid for the booking.

***

`checkout_date` <mark style="color:orange;">**string (Required)**</mark>

Check out date for the booking.

***

`number_of_infants` <mark style="color:orange;">**string (Required)**</mark>

Number of infants attached to the booking.

***

`listing_name` <mark style="color:orange;">**string (Optional)**</mark>

Name of the listing for which the booking was made.

***

`number_of_adults` <mark style="color:orange;">**string (Optional)**</mark>

Total number of adults attached to the booking.

***

`reservation_confirmation_code` <mark style="color:orange;">**string (Optional)**</mark>

Date when the reservation got confirmed.

***

`number_of_children` <mark style="color:orange;">**string (Required)**</mark>

Number of children attached to the booking.

***

`nights` <mark style="color:orange;">**string (Required)**</mark>

Total number of nights in the booking.

***

`listing_id` <mark style="color:orange;">**string (Required)**</mark>

Hotel id / Property id / listing id of the property allocated by OTA.

***

`checkin_date` <mark style="color:orange;">**string (Required)**</mark>

Date when the checkin will happen.

***

`number_of_guests` <mark style="color:orange;">**string (Required)**</mark>

Total number of guests.

### <mark style="color:yellow;">Response</mark>

{% hint style="info" %}
If acknowledgement is not received as required, system will try to repost the message 3 times. After 3 attempts, message will be removed from queue.

 "status": "Success"

#### <mark style="color:yellow;">Attributes</mark>

***

`hotelid / hotelcode` <mark style="color:orange;">**string (Required)**</mark>

Su property ID.

***

`channelcode` <mark style="color:orange;">**string (Required)**</mark>

Channel Code.&#x20;

{% hint style="info" %}
Accepted values must be either 19 or 244 or 9:

* Booking.com: 19
* Airbnb: 244
* Expedia: 9
  {% endhint %}

***

`message` <mark style="color:orange;">**string (Required)**</mark>

The message replied from property to OTA.&#x20;

{% hint style="info" %}
To use line breaker within message, **Airbnb supports: "\n"** and Booking.com **supports: "\\\n"**
{% endhint %}

***

`threadid` <mark style="color:orange;">**string (Required for Airbnb)**</mark>

Thread id of the conversation <mark style="color:yellow;">**(Required for Airbnb only)**</mark>

***

`guestid` <mark style="color:orange;">**string (Required for Airbnb)**</mark>

Guest id generated when communication occurs guest and property <mark style="color:yellow;">**(Required for Airbnb only)**</mark>

***

`bookingid` <mark style="color:orange;">**string (Required for Booking.com and Expedia)**</mark>

Booking id of the Booking

***

`listingid` <mark style="color:orange;">**string (Required)**</mark>

Hotel id / Property id / listing id of the property allocated by OTA.

***

`channelhotelid` <mark style="color:orange;">**string (Optional, applicable only for Airbnb)**</mark>

The channel hotel or host ID assigned to the property by the OTA.

***

}
