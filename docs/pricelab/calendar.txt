/calendar
Update rates and availability



This endpoint can update rates and availability of selected listings, along with the multi unit details of the listings.

listing_id should always be sent as a string, eg., "12345".

The first time a given date is entered in the data array for a listing it must contain both price, and available_units for the listing. The subsequent API calls can be a delta push or updates for specific date ranges.

If you need to price only one rate per listing, then you can ignore and skip the field rate_plan_id.

If you support multiple rate plans and need to update rates for all rate plans, then you should call the /rate_plans endpoint to first update the rate plan IDs and names. Further, you should call the /calendar endpoint mandatorily for default rate plan while providing the respective rate_plan_id of the default rate plan.
It is mandatory to set prices for the default rate plan using the /calendar API call.
