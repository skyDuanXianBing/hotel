POST
/calendar_trigger_url
Triggers a sync of complete calendar data FROM pms TO PriceLabs




This endpoint must be created BY THE INTEGRATION and the URL will be provided through the "calendar_trigger_url" in the /integration endpoint
The use of this endpoint signifies that the pms should utilize the /calendar endpoint to update the stored data in PriceLabs. This should be a complete data refresh for all tracked information within the date range

Parameters
Name	Description
X-SOURCE
string
(header)
Denotes the source of the request

X-SOURCE
X-PL-TIMESTAMP
string
(header)
The time stamp of when the request was created

X-PL-TIMESTAMP
X-PL-REQUESTID
string
(header)
uuid generated to id the request

X-PL-REQUESTID
X-PL-SIGNED-HEADERS
string
(header)
The signed combination of headers. 'X-SOURCE:X-PL-TIMESTAMP:X-PL-REQUESTID' signed with sha256 and the pms integration token. IF a version number is specified it is prepended before signing the header with 'VERSION' + ':' + header components. The signed value is then prepended with 'VERSION' + '.'

X-PL-SIGNED-HEADERS
X-PL-SIGNED-BODY
string
(header)
The signed combination of the final headers signature and body. 'X-PL-SIGNED-HEADERS' + 'request.body' signed with sha256 and the pms integration token

X-PL-SIGNED-BODY
calendar_trigger
object
(body)
start_date and end_date define inclusive range of necessary data
