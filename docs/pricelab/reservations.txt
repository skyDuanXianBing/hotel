POST
/reservations
Update property's reservations



This API will be used to keep the reservations for the listing up to date. Note that the start date and end date are synonymous with check in and check out dates for this API and that total_cost is the complete cost including taxes and fees. Note that listing_id should always be sent as a string, eg., "12345".
IMPORTANT Reservation IDs must be unique across all listings in the PMS. If they are not unique, please append a unique identifier (such as the listing ID) to the reservation ID before sending it to us and make sure to store this updated ID on your end for all future modifications to the same reservation.
Kindly note that once you implement this endpoint, you must push all reservations, including past(since 2020-01-01) and future reservations, to PriceLabs
