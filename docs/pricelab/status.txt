POST
/status
To fetch the current state of various objects



This API will be used to query PriceLabs for the current state of the various objects. A type and id parameter are passed to identify which records are needed. The id corresponds to the reservation_id or listing_id of the record.

Parameters
Name	Description
statuses *
object
(body)
Example Value
Model
{
  "statuses": [
    {
      "id": "string",
      "type": "calendar"
    }
  ]
}
Parameter content type

application/json
Responses
Response content type

application/json
Code	Description
200	
Request was processed, and response contains successes and failures of records.
Success will contain selected information from the queried record
Failure contains an error string describing the problem which halted processing, along with the item it was relevant to
