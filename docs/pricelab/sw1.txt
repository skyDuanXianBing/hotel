    ## Rate limiting & Timeout configuration
    
    Each partner token accessing the API has a limit of 300 requests per minute.

      * Retry-After: Seconds remaining before rate limit ends

We recommend setting the client timeout to 300 seconds to allow for any potential delays, even though our aim is to respond to all requests within that timeframe.</i>

        This endpoint can update integration specific details such as refreshing the API token if needed, or adjusting information that allows Pricelabs to send notices/updates.

         When you call <b>/integration</b>, please note that if <b><i>regenerate_token</i></b> is true, you will receive a new token in response which should be used to make subsequent API calls.
        
        <b>Delta Push</b>: If you wish to receive only delta updates for prices and restrictions, then you can do so by setting <b><i>delta_only</i></b> flag as true. Once the flag is set to true, you will start to receive only those prices and restrictions which have changed since the last sync. If at any point, you wish to pull full data, you can do so by calling the /get_prices endpoint with the attribute <b><i>delta_only</i></b> as true.
        
        <b>IMPORTANT</b>: The <i>features</i> array should be used to configure only those settings that the PMS intends to actively read from PriceLabs for all connected listings. 
        
          <i>Please note that if a setting is marked as <b>true</b>, it becomes visible to users in PriceLabs, who may enable it with the expectation that the PMS will consume and apply it. Therefore, make sure to enable only those settings that are actually supported and intended to be consumed from PriceLabs.</i>
          
          <b>Once the integration is live, any updates to the supported features must first be discussed with PriceLabs. This allows us to coordinate the release of new features to all users and ensure they function as expected.</b>
