# Channel Password Setup

### <mark style="color:yellow;">Endpoint</mark>

<mark style="color:green;background-color:green;">**`POST`**</mark>&#x20;

{% code overflow="wrap" %}

```url
https://connect.su-api.com/SUAPI/jservice/OTA_Setup
```

{% endcode %}

***

#### <mark style="color:yellow;">Attributes</mark>

`hotelid` <mark style="color:orange;">**alphanumeric (Required)**</mark>

Specifies the required node `hotelid` contains the IT provider's hotel id for which the provider wishes to add mapping information.

{% hint style="info" %}
`hotelid` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

***

`action` <mark style="color:orange;">**enum (Required)**</mark>

Indicates the call is meant to OTA setup for property.

{% hint style="info" %}
`action` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

{% hint style="info" %}
**Possible values:**

* `"setup"` → Performs both create and update operations. If the channel password does not exist in the system, it will create it; otherwise, it will update the existing one.
* `"remove"` → Directly performs a straightforward remove operation.
  {% endhint %}

***

`channelid` <mark style="color:orange;">**numeric (Required)**</mark>

Specifies the Channel Code for OTA in Su. For example, Air Asia (328), AltoVita, Rezlive (274).

{% hint style="info" %}
`channelid` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

***

`status` <mark style="color:orange;">**string (Required)**</mark>

Defines the status of the channel connection in the system.

{% hint style="info" %}
`status` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

{% hint style="info" %}
**Possible values:**

* `"Active"` → Channel integration is enabled
* `"Inactive"` → Channel integration is disabled
  {% endhint %}

***

`defaultmultiplier` <mark style="color:orange;">**number (Required)**</mark>

Defines the default multiplier value to be applied in rateplan mapping.

{% hint style="info" %}
`defaultmultiplier` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

***

`defaultsurcharge` <mark style="color:orange;">**number (Required)**</mark>

Specifies the default surcharge value to be applied in rateplan mapping.

{% hint style="info" %}
`defaultsurcharge` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

***

`currency` <mark style="color:orange;">**string (Required)**</mark>

Specifies the **currency code** to be used for channel pricing.

{% hint style="info" %}
`currency` is applicable for the channels: Air Asia (328), AltoVita (319), Dida Travel (320), Findbulous (300), happyeasygo (284), Hobse OLD (297), KliknBook (313), Omnihotelier (295), Rezlive (274), saffronstays (270), Simplotel (281), Tripfactory (296), VacanceSelect Travel (301)
{% endhint %}

***

`convertrates` <mark style="color:orange;">**boolean (Optional)**</mark>

Indicates whether the system should automatically **convert prices** for channel into the configured currency.

{% hint style="info" %}
`convertrates` is applicable for the channels: Air Asia (328), AltoVita (319), Dida Travel (320), Findbulous (300), happyeasygo (284), Hobse OLD (297), KliknBook (313), Omnihotelier (295), Rezlive (274), saffronstays (270), Simplotel (281), Tripfactory (296), VacanceSelect Travel (301)
{% endhint %}

***

`pricingmodel` <mark style="color:orange;">**string (Required)**</mark>

Defines the pricing model used by the channel integration.

{% hint style="info" %}
**Possible values:**

* `"OBP"` → Occupancy based Pricing
* `"RP"` → Room based Pricing
  {% endhint %}

{% hint style="info" %}
`pricingmodel` is applicable for Hyperguest OLD (291).
{% endhint %}

***

`infantage` <mark style="color:orange;">**number (Optional)**</mark>

Defines the maximum age limit considered as an infant.

{% hint style="info" %}
`infantage` is applicable for the channels: TXGB (317) and AltoVita (319)
{% endhint %}

***

`childage` <mark style="color:orange;">**number (Optional)**</mark>

Defines the maximum age limit considered as a child.

{% hint style="info" %}
`childage` is applicable for the channels: TXGB (317) and AltoVita (319)
{% endhint %}

***

`connection_name` <mark style="color:orange;">**string (Optional)**</mark>

Provides a **unique name** for the hotel connection to identify it easily in dashboards and reports.

{% hint style="info" %}
`connection_name` is applicable for all the channels listed [here](https://suissu.gitbook.io/su-api-documentation/su-channel-manager/channel-mapping-api-1/get-api-channel/channel-list-1/channel-list).
{% endhint %}

***

## <mark style="color:yellow;">**Sample Request**</mark>

```json
{
    "hotelid": "TestProp1",
    "action": "setup",
    "channelid": "328",
    "status": "Active",
    "defaultmultiplier": 1,
    "defaultsurcharge": 0,
    "currency": "AED",
    "convertrates": true,
    "pricingmodel": "OBP",
    "infantage": 10,
    "childage": 8,
    "connection_name": "test_connection_name"
}
```

***

#### <mark style="color:green;">Success Response - Body Elements</mark>

***

`Status` <mark style="color:orange;">**string**</mark>

Indicates the result of the API call.&#x20;

{% hint style="info" %}
Will be `"Success"` for successful operations.
{% endhint %}

***

`channelhotelid` <mark style="color:orange;">**integer**</mark>

Represents the unique hotel ID generated by the system, to be shared with the channel to establish the connection from their side.

***

#### <mark style="color:red;">Errors Response - Body Elements</mark>

`Status` <mark style="color:orange;">**string**</mark>

Indicates the result of the API call, which will be always `"Failed"` in case of an error response.

***

`Errors` <mark style="color:orange;">**array**</mark>

Array that contains details about any errors that occurred during the API call. Each object in the `Errors` array contains the following properties:

`Code` <mark style="color:orange;">**string**</mark>

Specific error code that helps identify the type of error.

{% hint style="info" %}
For example: `"400"` indicates a bad request.
{% endhint %}

`ShortText` <mark style="color:orange;">**string**</mark>

A short description of the error.
