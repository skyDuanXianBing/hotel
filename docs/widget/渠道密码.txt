# Channel Password Setup

**The Flow of the Channel Password Setup -**

Step 1 - Add Booking.com as a channel for which the Setup is to be done.

Step 2 - Add the valid data/values for multiple fields related to the Channel password setup process.&#x20;

Step 3 - Submit all the data. (If the channel password setup is successfully done, the system will acknowledge with a message - "Success". And in case of failure, system will acknowledge with an error message.)

***

#### <mark style="color:yellow;">Attributes</mark>

`hotelid` <mark style="color:orange;">**alphanumeric (Required)**</mark>

Specifies the required node `hotelid` contains the IT provider's hotel id for which the provider wishes to add mapping information.

***

`action` <mark style="color:orange;">**enum (Required)**</mark>

Indicates the call is meant to OTA setup for property.

{% hint style="info" %}
**Possible values:**

* `"setup"` → Performs both create and update operations. If the channel password does not exist in the system, it will create it; otherwise, it will update the existing one.
* `"remove"` → Directly performs a straightforward remove operation.
  {% endhint %}

***

`channelid` <mark style="color:orange;">**numeric (Required)**</mark>

Specifies the Channel Code for OTA in Su. (for example, Booking.com, Expedia, Airbnb).

***

`status` <mark style="color:orange;">**string (Required)**</mark>

Defines the status of the channel connection in the system.

{% hint style="info" %}
**Possible values:**

* `"Active"` → Channel integration is enabled
* `"Inactive"` → Channel integration is disabled
  {% endhint %}

***

`channelhotelid` <mark style="color:orange;">**alphanumeric (Required)**</mark>

Specifies the **unique hotel ID** used in the external distribution channel or PMS (Property Management System).

***

`pricingmodel` <mark style="color:orange;">**string (Optional)**</mark>

Defines the **pricing model** used for the hotel.

{% hint style="info" %}
**Possible values:**

* `"OBP"` → Occupancy based Pricing
* `"RP"` → Room based Pricing
  {% endhint %}

***

`currency` <mark style="color:orange;">**string (Required)**</mark>

Specifies the **default currency code** used for pricing.

***

`convertrates` <mark style="color:orange;">**boolean (Optional)**</mark>

Indicates whether the API should automatically **convert prices** into the configured currency.

***

`defaultmultiplier` <mark style="color:orange;">**number (Required)**</mark>

Defines the default multiplier value applied in rateplan mapping.

***

`defaultsurcharge` <mark style="color:orange;">**number (Required)**</mark>

Specifies the default surcharge value applied in rateplan mapping.

***

`override_pms_auth` <mark style="color:orange;">**integer (Optional)**</mark>

When enabled, STAAH credentials will override the PMS credentials used to push updates to OTAs. This parameter is applicable to properties that have enabled STAAH as their channel manager while the PMS maintains its own OTA credentials.

{% hint style="info" %}

**Possible Values:**

* `0` – Do not override PMS authorization
* `1` – Override PMS authorization using STAAH credentials
  {% endhint %}

***

`connection_name` <mark style="color:orange;">**string (Optional)**</mark>

Provides a **unique name** for the hotel connection to identify it easily in dashboards and reports.

***

## <mark style="color:yellow;">**Sample Request**</mark>

```json
{
  "hotelid": "AWSTEST",
  "action": "setup",
  "channelid": 19,
  "status": "Active",
  "channelhotelid": "test_test",
  "pricingmodel": "OBP",
  "currency": "USD",
  "convertrates": true,
  "defaultmultiplier": 1,
  "defaultsurcharge": 0,
  "override_pms_auth": 0,
  "connection_name": "test_connection_name"
}
```

***

#### <mark style="color:green;">Success Response - 200 Body Elements</mark>&#x20;

`channelmapping` <mark style="color:orange;">**arrray of objects**</mark>

Channel mapping response.

***

`type` <mark style="color:orange;">**string**</mark>

Managable or non-managable.

***

`rateplan` <mark style="color:orange;">**string**</mark>

The rate plan name.

***

`channelroomname` <mark style="color:orange;">**string**</mark>

Indicates channel room name.

***

`channelroomid` <mark style="color:orange;">**string**</mark>

Indicates channel room id.

***

`channelratename` <mark style="color:orange;">**string**</mark>

Indicates channel rate name.

***

`channelrateid` <mark style="color:orange;">**alphanumeric**</mark>

Indicates channel rate id.

***

`mappingstring` <mark style="color:orange;">**string**</mark>

Indicates channel room id and rate id.

***

`Status` <mark style="color:orange;">**string**</mark>

Indicates the result of the API call.&#x20;

{% hint style="info" %}
Will be `"Success"` for successful operations.
{% endhint %}

***

#### <mark style="color:red;">Errors Response - 400 Body Elements</mark>

`Errors` <mark style="color:orange;">**array**</mark>

Array that contains details about any errors that occurred during the API call. Each object in the `Errors` array contains the following properties:

`Code` <mark style="color:orange;">**string**</mark>

Specific error code that helps identify the type of error.

{% hint style="info" %}
For example: `"400"` indicates a bad request.
{% endhint %}

`ShortText` <mark style="color:orange;">**string**</mark>

A short description of the error.

***

`Status` <mark style="color:orange;">**string**</mark>

Indicates the result of the API call, which will be always `"Failed"` in case of an error response.
