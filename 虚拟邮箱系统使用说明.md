# 虚拟邮箱系统使用说明

## 📌 系统简介

本系统通过**虚拟邮箱**实现与客人的邮件沟通,前台员工在PMS系统中看到的是**聊天对话框**,而客人收到的是**普通邮件**。

---

## 🎯 核心功能

### 1️⃣ 自动创建专属邮箱
- 每个订单自动生成一个虚拟邮箱
- 格式: `order-订单号@酒店域名.jp`
- 示例: `order-RSV1737370001234@myhotel.smartorder.jp`

### 2️⃣ 前台统一管理
- 所有客人沟通集中在PMS"收件箱"
- 界面类似微信聊天,简单易用
- 无需切换邮箱软件

### 3️⃣ 自动发送通知
- 订单确认时自动发送欢迎消息
- 入住前24小时发送入住指引
- 入住当天发送房间信息
- 退房后发送感谢消息

## ⚙️ 自动化消息设置

### 可设置的触发时机:
1. **订单确认时** - 立即发送欢迎消息
2. **入住前24小时** - 发送入住指引
3. **入住当天** - 发送房间信息和密码
4. **退房当天** - 发送退房提醒
5. **退房后** - 发送感谢和评价邀请

### 设置方法:
1. 进入 PMS → 设置 → 通用设置 → 自动化消息
2. 点击"新建模板"
3. 选择触发时机 (如: 订单确认时)
4. 输入消息内容 (支持变量: {客人姓名}、{房间号}等)
5. 选择适用渠道 (全部/Booking.com/Airbnb等)
6. 启用模板

## 🔄 日本预订平台对接

### 订单自动同步
```
客人在日本平台预订
    ↓
自动推送到PMS
    ↓
PMS创建订单
    ↓
自动生成虚拟邮箱
    ↓
自动发送欢迎邮件
```

### 价格库存双向同步
- PMS修改房价 → 自动同步到日本平台
- PMS修改库存 → 自动同步到日本平台
- 日本平台订单 → 自动扣减PMS库存

## 💬 常见问题

**Q: 客人回复邮件后,多久能在PMS中看到?**
A: 约30秒-1分钟,系统每30秒自动检查新邮件

**Q: 员工发送消息后,客人多久能收到邮件?**
A: 约30秒-1分钟,通过邮件服务器发送

**Q: 如果客人使用其他邮箱回复会怎样?**
A: 只要回复到虚拟邮箱地址,都会正常显示在PMS中

**Q: 订单完成后,虚拟邮箱还能用吗?**
A: 可以手动关闭会话,关闭后不再接收新邮件

**Q: 可以群发消息吗?**
A: 可以,选择多个会话,批量发送通知

**Q: 支持发送图片或附件吗?**
A: 目前支持纯文本,未来版本会支持附件
